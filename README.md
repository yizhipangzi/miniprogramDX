# 东京电影院（微信小程序）

通过东京地图选择区域 → 查看该区域的电影 / 电影院 → 查看排片场次。

---

## 一、页面与功能式样

### 1. 首页 `pages/index`（地图选区）
- 展示东京地图底图 `images/tokyo_map.png`（原图 853 × 1844）。
- 图上叠加 9 个**透明热区按钮**，坐标按原图像素定义、随屏幕等比缩放（见 `index.js` 的 `SPOTS`）。
- 点击热区 → 红色高亮动效 → 跳转到对应区域页 `pages/district`。
- 9 个区域 id：`takadanobaba / ikebukuro / shinjuku / ueno / suidobashi / shibuya / yurakucho / roppongi / ebisu`
  - ⚠️ 注意：地图标注里的 `iidabashi`、`ginza` 对应数据/导航 id 为 `suidobashi`、`yurakucho`（保持与数据文件名一致，否则无法加载）。

### 2. 区域页 `pages/district`（双视图）
右上角图标切换两种索引模式：

| 图标 | 模式 | 说明 |
|------|------|------|
| 🎞️ | **电影索引**（默认） | 该区域所有电影**去重聚合**，按评分从高到低排序。点卡片展开 → 该片在各影院的**场次按时间排序**。 |
| 🏢 | **电影院索引** | 该区域电影院列表，点击进入 `pages/theater` 看单影院排片。 |

- **评分块**（卡片左侧彩色块）：点击弹出**电影详情浮层**（见下）。
- **场次行**：点击复制该影院 eiga.com 链接（小程序不能直接跳第三方网页，故复制到剪贴板）。

### 3. 电影详情浮层（区域页内）
比页面小一圈的居中浮层，点遮罩 / ✕ 关闭。内容式样：

```
┌─────────────────────────────┐
│ [封面图]  标题(日)            │
│          标题(中)            │
│          年份 · 国家          │
├─────────────────────────────┤
│ ⭐ 8.4  某瓣  / 536看过 1587想看 │
│ 👤 导演  xxx                 │
│ 🎭 主演  xxx                 │
│ 🏷️ 类型  剧情 / 爱情          │
├─────────────────────────────┤
│ 📰 某瓣短评                  │
│ xxxxxxxxxxxxxxxxx           │
└─────────────────────────────┘
```
- 每一行**有数据才显示，缺失自动隐藏**。
- 「看过 / 想看」仅在有某瓣分时显示。

### 4. 影院页 `pages/theater`（单影院排片）
- 某电影院的上映电影列表，按评分排序，每部带场次时间。

## 二、数据结构（现状）

数据按区域拆分为 9 个文件：`data/{区域id}.js`，每个 `module.exports = { id, name, theaters: [...] }`。

```
district
├─ id            区域 id（= 文件名，导航用）
├─ name          区域显示名
└─ theaters[]
   ├─ id         eiga 影院路径，如 "13/130611/3071"（用于拼 eiga 影院页 URL）
   ├─ name       影院名
   └─ movies[]
      ├─ id              eiga 电影 id（用于拼 eiga 电影页/封面）
      ├─ titleJp         日文片名
      ├─ titleCn         中文片名
      ├─ doubanScore     某瓣评分(0-10)
      ├─ doubanVotes     某瓣评价人数
      ├─ eigaRating      eiga评分(0-5)
      ├─ eigaRatingCount eiga评价人数
      ├─ genre           类型，如 "剧情 / 爱情"
      ├─ duration        时长(分钟)
      ├─ year            年份
      ├─ country         国家/地区
      ├─ director        导演 ★字段存在但全部为 null
      └─ sortScore       排序用分数
```

**封面图**：后端 JSON 每个 `movie` 直接带 `poster` 字段（完整 URL）。由 pipeline 生成：step1 从 eiga 海报页 `/movie/{id}/photo/` 的 `.movie-photo` 抓取（升到高清 `/640.jpg`），eiga 没有海报时 step2 回退抓某瓣 `#mainpic`，step4 合成为最终 `poster`（eiga 优先）。

## 三、字段填充率（783 条电影记录）

| 字段 | 填充率 | 状态 |
|------|--------|------|
| titleJp | 100% | ✅ |
| titleCn | 91% | ✅ |
| genre（类型） | ~100% | ✅ |
| year / country / duration | ~100% | ✅ |
| eigaRating | 77% | ✅ |
| doubanScore | 49% | ⚠️ 半数缺某瓣分 |
| **director（导演）** | **0%** | ❌ 字段在但全为 null |
| 封面图（poster 字段） | — | ✅ step1 eiga 海报，缺失回退 step2 某瓣海报 |

## 四、暂时不足、需要补充的数据

详情浮层已预留以下字段，**数据补上即自动显示**（字段名为 `movie` 对象上应新增的键）：

| 浮层显示项 | 需要的字段名 | 现状 | 建议来源 |
|-----------|-------------|------|---------|
| 👤 导演 | `director` | ❌ 全 null | 某瓣 / eiga 详情页 |
| 🎭 主演 | `cast` | ❌ 不存在 | 某瓣 / eiga 详情页（建议存字符串，如 "梁朝伟 / 张曼玉"） |
| 看过人数 | `doubanWatched` | ❌ 不存在 | 某瓣（数字） |
| 想看人数 | `doubanWish` | ❌ 不存在 | 某瓣（数字） |
| 📰 某瓣短评 | `doubanComment` | ❌ 不存在 | 某瓣热门短评（字符串，建议 1 条） |

### 其他需要补充 / 待改进的数据

1. **真实排片场次** ❌（当前为程序生成的假数据）
   - 现在场次由 `genShowtimes()` 根据「影院+电影」稳定生成，**非真实时刻表**。
   - 真实数据建议结构：每个 `movie` 下加 `showtimes`，按影院给出当天场次，例如：
     ```js
     showtimes: [
       { theaterId: "13/130611/3071", times: ["10:10", "13:55", "19:15"] }
     ]
     ```
   - 接入后替换 `district.js` / `theater.js` 里的 `genShowtimes` 即可。

2. **某瓣评分覆盖** ⚠️ 仅 49%，可补全另一半。

3. **某瓣精确链接** ⚠️ 当前无某瓣 subject id，外链用的是「片名搜索」。若补 `doubanId`，可改为精确页 `movie.douban.com/subject/{doubanId}/`。

## 五、数据维护

- **更新封面**：由 pipeline 抓取（step1 eiga 海报页 / step2 某瓣 `#mainpic`），step4 导出到各区 JSON 的 `poster` 字段，无需单独脚本。
- **数据拆分**：`data/{区域}.js` 为运行时实际加载文件（真机 `require` JSON 不稳定，故用 `.js` 模块导出）。

---

## 六、已知平台限制

- **不能直接打开第三方网页**：小程序 `web-view` 只能加载已备案的自有业务域名，eiga.com / 某瓣无法配置；故所有「外链」均为**复制链接到剪贴板**，由用户粘贴到浏览器打开。
- **不能在客户端爬取网页**：`wx.request` 仅限已配置的业务域名，无法抓取 eiga HTML，故封面采用**构建期脚本预抓取**。
- **外链图片可正常显示**：`<image>` 组件加载外网图片不受合法域名限制，`media.eiga.com` 封面真机可显示。
- **包体积 2MB 上限**：`images/tokyo_map.png` 约 2.3MB **已超限**，需压缩或改用 CDN 网络图后才能正常发布（详见地图图片优化）。
